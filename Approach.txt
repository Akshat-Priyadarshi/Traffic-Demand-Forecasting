Approach and System Architecture

This solution prioritizes system reliability, robust spatial-temporal forecasting, and strict adherence to the provided competition data. The architecture relies on physical geographic clustering and autoregressive momentum, utilizing strict Out-of-Fold (OOF) validation to prevent target leakage and maximize generalization on unseen test data.

Key Engineering Highlights:
1. Spatial Decoding & Neighborhood Clustering: `pygeohash` was utilized to extract precise Latitude and Longitude coordinates from the provided geohash strings. These coordinates were then grouped into 50 distinct "Traffic Zones" using K-Means Clustering, providing the tree algorithms with highly efficient geographic neighborhood boundaries.
2. Continuous Time & Autoregressive Momentum: Time was modeled as a continuous cycle using sine and cosine transformations. Furthermore, to capture traffic momentum, the training data was mathematically shifted to engineer explicit 24-hour and 7-day historical lag features, granting the models memory of recent demand at exact locations.
3. Leak-Proof Target Encoding (The Keystone): To establish historical traffic baselines without triggering target leakage, Scikit-Learn's Target Encoding was applied to the geohashes strictly *inside* the K-Fold cross-validation boundaries. This maps explicit historical demand averages safely to the unseen test set.
4. Hyper-Stable Ensemble Engine: A 5-Fold Cross-Validation strategy trained a blended ensemble of LightGBM and CatBoost Regressors. CatBoost natively handles high-cardinality categoricals, while LightGBM rapidly optimizes the continuous spatial and clustered floats. A robust 50/50 uniform blend prevents the validation overfitting often caused by automated weighting optimizers.

Data Integrity Note:
During the exploratory phase, I identified that this dataset aligns with the 2019 Grab AI open-source dataset. While it is mathematically possible to achieve a near-perfect R2 score by performing a direct key-intersection with the historical Kaggle data, my team and I have strictly avoided using any external data to prevent data leakage. The submitted predictions (~91.65% R2) are generated entirely via our custom machine learning ensemble using only the provided `train.csv`, preserving the absolute integrity of the evaluation metric.
